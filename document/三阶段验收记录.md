## 各任务详情

### 任务 A — 收口桌面端启动与连接主流程

- **输入锁定**：连接后，服务器地址和用户 ID 输入框自动变为不可编辑（`DISABLED`）；断开后恢复。
- **按钮状态一致性**：新增 `_set_connection_ui_state(*, connecting: bool)` 统一方法，管理连接/断开按钮、密钥按钮的启用/禁用。
- **密钥保护**：已连接状态下禁止重新生成或加载密钥，弹窗提示"已连接时不允许修改密钥"。
- **断连清理**：断开连接时清空联系人列表、重置当前聊天对象、恢复所有 UI 控件初始状态。
- **Entry 控件引用**：`_build_login_area` 和 `_build_key_area` 存储 Entry/Button 引用，供状态管理使用。

### 任务 B — 收口联系人、公钥与会话状态

- **指纹展示**：联系人列表下方新增指纹标签（`_contact_fp_var`），选择联系人时实时显示其公钥指纹。
- **公钥状态提示**：选中联系人但尚未收到其公钥时，聊天区标签显示"等待公钥..."。
- **下线检测**：`_handle_user_list` 对比在线列表，若当前聊天对象已下线，在聊天区插入 `--- {peer} 已下线 ---` 系统消息，并更新聊天标签。
- **公钥到达更新**：`_handle_incoming_public_key` 在收到公钥后，若发送者是当前聊天对象，立即更新聊天标签和指纹显示。

### 任务 C — 收口消息发送、自动解密与聊天显示

- **发送侧分步日志**（截图安全，不记录明文内容）：
  - ① 准备发送消息（明文长度）
  - ② 生成一次性 AES-256 会话密钥
  - ③ AES-GCM 加密完成 → 密文长度、nonce 长度
  - ④ RSA 公钥加密会话密钥 → wrapped_key 长度（对方指纹）
  - ⑤ 密文消息包已发送（服务端仅可见密文，无法还原明文）
- **接收侧分步日志**：
  - ① 收到来自 {sender} 的密文消息包
  - ② wrapped_key 长度、密文长度
  - ③ RSA 私钥解包 wrapped_key → 恢复 AES 会话密钥成功
  - ④ AES-GCM 解密成功 → 明文长度
- **`message_crypto.py` 增强**：
  - `encrypt_chat_message` 调试字典新增 `wrapped_key_length`、`nonce_length`、`session_key_bits`。
  - `decrypt_chat_message` 调试字典新增 `wrapped_key_length`、`nonce_length`。

### 任务 D — 完善 Crypto Console 与安全状态展示

- **彩色标签系统**：`_build_crypto_console` 中配置 8 种颜色标签：
  | 标签名 | 颜色 | 用途 |
  |--------|------|------|
  | `log_encrypt` | #0066cc（蓝） | 加密流程日志 |
  | `log_decrypt` | #009933（绿） | 解密流程日志 |
  | `log_key` | #996600（棕） | 密钥操作日志 |
  | `log_send` | #0066cc（蓝） | 发送日志 |
  | `log_recv` | #009933（绿） | 接收日志 |
  | `log_conn` | #666666（灰） | 连接状态日志 |
  | `log_error` | #cc0000（红） | 错误日志 |
  | `log_warn` | #cc6600（橙） | 警告日志 |
- **`_append_crypto_log(text, tag="")`**：支持可选 `tag` 参数，所有调用处均传入语义化标签。
- **状态栏增强**：连接成功时显示 `已连接 | 用户: {uid} | 本地指纹: {fp}`。
- **断连事件处理**：`EVT_DISCONNECTED` 触发完整状态重置（UI 控件、联系人列表、当前会话），并显示断连原因。

## 已知限制 / 后续改进方向

- 聊天记录不持久化，关闭窗口即丢失。
- 不支持群聊，仅限一对一。
- 公钥分发依赖服务端转发，无独立 PKI 或证书链验证。
- 未做 Windows 打包（.exe），当前需 Python 环境运行。
- Wireshark 抓包展示、最终报告和 PPT 属于后续交付包装阶段。
