## 幻灯片详细内容

### 幻灯片 1：封面

- 课题名称：端到端加密即时通讯软件设计与实现
- 课程名称、姓名、学号、指导教师、日期

### 幻灯片 2：项目背景与目标（1 min）

- **背景**：即时通讯安全需求日益增长，传统中间人可窃听的通信架构存在固有缺陷
- **目标**：实现端到端加密（E2EE）聊天系统，消息全链路密文，服务端无法解密
- **技术栈一句话**：Python 3.12 + websockets + cryptography

### 幻灯片 3：对称加密 — AES-256-GCM（1 min）

- AES-256 密钥长度 256 位，GCM 模式 = AEAD（加密 + 完整性）
- 12 字节随机 Nonce，128-bit Auth Tag
- 优势：无需额外 HMAC，性能优秀
- 与 AES-CBC 对比（可选表格）

### 幻灯片 4：非对称加密 — RSA-2048 OAEP（1 min）

- RSA-2048 公私钥对，OAEP 填充（SHA-256, MGF1-SHA-256）
- 用途：不直接加密消息（有长度限制），而是加密 AES 会话密钥
- 公钥交换：注册时上传，服务端广播分发

### 幻灯片 5：混合加密流程图（0.5 min）

- **加密**：明文 → 生成一次性 AES 密钥 → AES-GCM 加密 → RSA-OAEP 包裹密钥 → 发送
- **解密**：接收 → RSA 解包密钥 → AES-GCM 解密 → 明文
- 配图：加密解密流程示意图（可用 `Demonstration/` 下的流程图）

### 幻灯片 6：盲转发与安全模型（0.5 min）

- 服务端 = 盲转发节点，不持有任何私钥
- 即使攻击者控制服务端，也无法获取明文
- 每条消息独立密钥 → 前向安全性

### 幻灯片 7：系统架构图（1 min）

- 五层架构图：加密层 → 协议层 → 网络层 → 会话层 → 表现层
- 客户端 ↔ WebSocket ↔ 中继服务端 ↔ WebSocket ↔ 客户端
- 可复用 `Demonstration/` 下的架构图

### 幻灯片 8：通信协议设计（1 min）

- 自定义 JSON 协议：`type` + `payload`
- 消息类型列表：`register`、`user_list`、`chat_message`、`file_transfer` 等
- 协议版本化设计（v2 向后兼容）
- 密文字段结构：`wrapped_key` + `nonce` + `ciphertext`（均 Base64）

### 幻灯片 9：核心模块划分（0.5 min）

| 消息加密 | `message_crypto.py`   | 混合加密/解密入口           |

## 现场实操演示（3 min）

### 准备

- 提前打开 3 个终端窗口（或使用 VSCode 分屏终端）
- 可选：打开 Wireshark，过滤 `tcp.port == 8765`

### 演示流程

| 步骤 | 时间 | 操作                                | 讲解要点                                          |
| ---- | ---- | ----------------------------------- | ------------------------------------------------- |
| 1    | 20s  | `python chat_server.py`             | "服务端启动，注意日志格式"                        |
| 2    | 30s  | 启动 Alice 客户端 → 生成密钥 → 连接 | "密钥指纹显示在 Crypto Console"                   |
| 3    | 30s  | 启动 Bob 客户端 → 连接              | "联系人列表自动出现 Alice，公钥已交换"            |
| 4    | 40s  | Alice 发送 "Hello Bob"              | **核心展示**：切到 Crypto Console 看加密 5 步日志 |
| 5    | 20s  | 切到 Bob → 看到解密消息             | 切到 Crypto Console 看解密 4 步日志               |
| 6    | 20s  | 切到服务端终端                      | "只有类型和 payload 长度，没有明文"               |
| 7    | 20s  | Alice 发送一张图片                  | "文件也经过 E2EE 加密传输"                        |

### 演示备注

- 如果时间紧张，跳过步骤 7（文件传输）
- 如果老师要求看 Web 端，可快速打开浏览器展示
- 演示前确保服务端端口 8765 未被占用
