## 各任务详情

### 任务 A — 收敛消息协议 (`chat_protocol.py`)

- **模块文档**：新增完整的 payload schema 文档字符串，列出每种消息类型的必填字段。
- **`_PAYLOAD_REQUIRED_FIELDS`**：字典映射消息类型 → 必填 payload 字段元组。
- **`_REQUIRES_RECEIVER`**：需要 `receiver_id` 的消息类型集合（`public_key`、`chat_message`、`ack`）。
- **`parse_message()` 增强**：
  - 新增 `strict_payload: bool = True` 参数。
  - 校验 payload 必须为 dict。
  - 校验 `_REQUIRES_RECEIVER` 中的类型必须携带非空 `receiver_id`。
  - 严格模式下校验必填 payload 字段。

### 任务 B — 固化中继服务端行为 (`chat_server.py`)

- 全面重写 `ChatServer` 类。
- **不活跃清理**：`_last_active` 字典 + `_inactive_checker()` 协程，每 30s 扫描一次，超时（默认 120s）自动断开。
- **重复注册**：同一 `user_id` 再次注册时关闭旧 WebSocket 连接。
- **未注册拒绝**：非 `register` 类型消息若来自未注册客户端，返回 error。
- **离线发送提示**：`chat_message` 的接收方不在线时，向发送方返回 error。
- **CLI 参数**：新增 `--timeout` 命令行参数。

### 任务 C — 完善心跳与连接状态

- **服务端**：`handle_connection` 中收到 `heartbeat` 后调用 `_touch()` 刷新活跃时间。
- **客户端**：`HEARTBEAT_INTERVAL = 30`，`_heartbeat_loop()` 异步方法定期发送心跳；`send_heartbeat()` 公开方法。

### 任务 D — 整理客户端网络事件模型

- **`chat_client.py`**：
  - 定义事件类型常量：`EVT_CONNECTED`、`EVT_DISCONNECTED`、`EVT_CHAT_MESSAGE`、`EVT_PUBLIC_KEY`、`EVT_USER_LIST`、`EVT_ACK`、`EVT_ERROR`。
  - `_MAX_EVENT_QUEUE_SIZE = 1000`，`_put_event()` 处理队列满时丢弃最旧事件。
  - `_async_main` 启动 heartbeat_task 与 recv/send 并行。
  - 所有字符串事件类型替换为常量。
  - 断连事件包含 `"reason"` 字段。
- **`desktop_chat_gui.py`**：`_handle_network_event` 使用 `EVT_*` 常量。
