# 最低合规版人工验收手册

本文档描述从零开始启动并完成一次完整的双客户端端到端加密聊天演示的全部步骤。
按本文档操作可独立复现实验结果，无需额外口头说明。

---

## 前置条件

| 项目     | 要求                                  |
| -------- | ------------------------------------- |
| Python   | ≥ 3.10                                |
| 依赖包   | `pip install websockets cryptography` |
| 操作系统 | Windows / macOS / Linux 均可          |
| 终端窗口 | 至少 3 个（1 个服务端 + 2 个客户端）  |

确认依赖已安装：

```bash
pip install -r requirements.txt
```

---

## 第一步：启动中继服务端

在项目根目录下打开第 1 个终端：

```bash
python chat_server.py --host 127.0.0.1 --port 8765
```

预期输出：

```
[HH:MM:SS] INFO 服务端启动: ws://127.0.0.1:8765 (不活跃超时: 120s)
```

> **关键验证点**：服务端日志在整个演示过程中**不会出现任何聊天明文**，只会显示消息类型、发送方、接收方和 payload 长度。

---

## 第二步：启动客户端 A（Alice）

在第 2 个终端中：

```bash
python desktop_chat_gui.py
```

在 GUI 中执行以下操作：

1. **生成密钥**：点击"生成密钥"按钮。
   - 预期：Crypto Console 显示 `[密钥] 已生成 RSA-2048 密钥对，本地公钥指纹: xxxxxxxx`。
   - 预期：状态栏显示 `密钥已生成 (指纹: xxxxxxxx)`。

2. **输入连接信息**：
   - 服务器地址保持默认 `ws://127.0.0.1:8765`。
   - 用户 ID 输入 `alice`。

3. **连接服务端**：点击"连接"按钮。
   - 预期：状态栏显示 `已连接 | 用户: alice | 本地指纹: xxxxxxxx`。
   - 预期：Crypto Console 显示 `[连接] 已成功连接到服务端并完成注册。`
   - 预期：服务器地址和用户 ID 输入框变为不可编辑（灰色）。

---

## 第三步：启动客户端 B（Bob）

在第 3 个终端中：

```bash
python desktop_chat_gui.py
```

重复与 Alice 相同的流程，但用户 ID 输入 `bob`。

连接成功后：

- **Alice 和 Bob 的 GUI** 左侧联系人列表中应各自看到对方。
- **Crypto Console** 应显示 `[密钥] 自动导入 xxx 的公钥 (指纹: xxxxxxxx)`。

> **关键验证点**：双方均可在联系人列表中看到对方，且自动获取了对方公钥。

---

## 第四步：Alice 向 Bob 发送加密消息

在 **Alice** 的 GUI 中：

1. 点击左侧联系人列表中的 `bob`。
   - 预期：聊天区顶部显示 `与 bob 聊天  (公钥指纹: xxxxxxxx)`。

2. 在输入框中输入：`你好，这是一条加密消息！`

3. 点击"发送"或按回车。

**Alice 侧预期结果**：

- **聊天区**：显示 `[HH:MM:SS] 我: 你好，这是一条加密消息！`（蓝色）。
- **Crypto Console** 完整加密链路日志：
  ```
  [加密] ① 准备发送消息 (明文长度: 12 字符)
  [加密] ② 生成一次性 AES-256 会话密钥
  [加密] ③ AES-GCM 加密完成 → 密文长度: xx, nonce 长度: xx
  [加密] ④ RSA 公钥加密会话密钥 → wrapped_key 长度: xx (对方指纹: xxxxxxxx)
  [发送] ⑤ 密文消息包已发送给 bob (服务端仅可见密文，无法还原明文)
  ```

**Bob 侧预期结果**：

- **聊天区**：显示 `[HH:MM:SS] alice: 你好，这是一条加密消息！`（绿色）。
- **Crypto Console** 完整解密链路日志：
  ```
  [接收] ① 收到来自 alice 的密文消息包
  [接收] ② wrapped_key 长度: xxx, 密文长度: xxx
  [解密] ③ RSA 私钥解包 wrapped_key → 恢复 AES 会话密钥成功
  [解密] ④ AES-GCM 解密成功 → 明文长度: 12 字符
  ```

**服务端日志预期**（检查第 1 个终端）：

```
[HH:MM:SS] INFO 收到消息: type=chat_message sender=alice receiver=bob payload_len=xxx
```

> **关键验证点**：服务端日志中只有 `payload_len`（长度），**没有任何明文内容**。

---

## 第五步：Bob 回复 Alice

在 **Bob** 的 GUI 中选择 `alice`，输入 `收到！加密通信正常。` 并发送。

验证双方聊天区和 Crypto Console 显示正确。

---

## 第六步：异常场景演示

### 6.1 未选联系人时发送

- 不选择任何联系人，直接输入消息并发送。
- 预期：弹窗提示"请先选择一个联系人。"

### 6.2 未生成密钥时连接

- 新开一个客户端，不点"生成密钥"直接点"连接"。
- 预期：弹窗提示"请先生成或加载密钥，再连接服务端。"

### 6.3 对方下线后发送

- 关闭 Bob 的客户端窗口。
- 在 Alice 侧向 bob 发送消息。
- 预期：Alice 的 Crypto Console 显示 `[错误] 用户 bob 不在线。`
- 预期：Alice 的聊天区显示 `--- bob 已下线 ---`。

### 6.4 断开后重连

- Alice 点击"断开"，再次点击"连接"。
- 预期：成功重新连接，状态栏恢复为已连接状态。

```bash

python chat_server.py

# 终端 2 — 客户端 A
python desktop_chat_gui.py

# 终端 3 — 客户端 B
python desktop_chat_gui.py
```

- wrapped_key 长度
- 解密成功

在客户端 B 中向 alice 发送消息，验证双向通信正常。

检查服务端终端日志：
