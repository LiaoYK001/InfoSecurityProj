## 4. 五阶段开发顺序

### 任务 A：采集运行截图

目标：按截图清单完成所有 GUI、服务端、抓包截图。

必须完成：

- 启动服务端 + 双客户端，完成一次完整加密聊天演示。
- 按 `document/week8_demo_script.md` 中的截图清单逐项截图。
- 启动 Wireshark，过滤 `tcp.port == 8765`，捕获聊天过程并截图。
- 截图保存到 `document/screenshots/` 目录（或报告中直接插入）。

完成标准：

- 截图清单中的所有项均已采集。
- 抓包截图包含过滤条件、帧列表和展开的 payload。

### 任务 B：填充报告正文

目标：按 `document/report_outline.md` 骨架逐章填充正文。

必须完成：

- 第一章（任务分析）：基于 `NECESSARY_REQUIREMENT.md` 填写。
- 第二章（理论基础）：基于 `document/design_decisions.md` 中的算法选型理由扩写。
- 第三章（设计）：参考 `document/design_decisions.md` 和代码结构，补写概要设计图和详细设计时序图。
- 第四章（实现与测试）：参考 `document/result_note.md` 填写测试结果。
- 第五章（结果分析）：直接引用 `document/result_note.md` 内容。
- 第六章（截图）：插入任务 A 采集的截图。
- 第七章（使用说明）：参考 `README.md` 填写。
- 第八章（团队分工）：参考 `document/team_division.md` 填写。

完成标准：

- 报告所有章节均有实质内容，不存在空白占位。
- 报告格式规范（Word 或 PDF）。

### 任务 C：代码注释自查

目标：确保源码注释满足课程红线要求。

必须完成：

- 逐文件检查核心 `.py` 文件是否有充分注释。
- 特别关注：`message_crypto.py`（加密链路注释）、`chat_server.py`（盲转发说明）、`desktop_chat_gui.py`（UI 逻辑注释）。
- 补充缺失的关键注释（不过度注释，只补必要的）。

完成标准：

- 每个核心函数和类有文档字符串或行内注释。
- 满足"充分注释，否则视为抄袭"的课程要求。

### 任务 D：打包压缩与提交

目标：完成最终提交动作。

必须完成：

- 按 `document/submission_checklist.md` 逐项勾选确认。
- 清理不必要的临时文件（`build/`、`__pycache__/`、`.spec` 文件等）。
- 按"第X组+题目"命名规范打包为压缩包。
- 压缩包内容：最终报告 + 程序源码 + 可运行程序 + 实验结果说明。
- 发送至 `xiongf@bjtu.edu.cn`。

完成标准：

- 压缩包命名正确。
- 压缩包内容完整。
- 已成功发送。

### 任务 E：汇报演练

目标：按脚本完成一次 15 分钟汇报演练。

必须完成：

- 按 `document/week8_demo_script.md` 从头到尾走一遍。
- 计时确认 15 分钟以内。
- 记录演练中发现的问题并修正。

完成标准：

- 可以在 15 分钟内完成理论、代码、演示三部分。
- 对常见提问有准备好的回答。
